# Production Acceptance Evidence Guide

Status: **BLOCKED before live authorization**.

Use the reserved screenshot names in `matrix.md`, captured in numeric order. Start `00-cross-turn-workflow.mp4` before the first visible action and keep it continuous across the RESEARCH/ASK/suggestion restart-and-delivery journey. Pause or obscure capture only while entering a password, OTP, or secret; never record those values.

Each row needs three independent elements:

1. the natural visible action on the deployed site or real X profile;
2. the numbered screenshot and, where cross-turn behavior matters, a video timestamp;
3. a read-only receipt from the independent state/log/provider/profile surface after the visible action.

Before recording, replace every placeholder with the verified deployed SHA, deployment id, URL, rulebook hash/version, test ids, approval receipts, rollback target, bounded budget, and cleanup commands. Do not create empty screenshot/video placeholders. Missing access or approval is BLOCKED.

After the run, inspect every image for location, target, and visible state; inspect the entire video; then complete `cleanup.md`. A weak or missing artifact is FAIL/BLOCKED, never an inferred PASS.
